# Outline for Final Written Paper

## 1. Introduction
- Brief context: need for multilingual real-time communication.
- Purpose of the project: enable seamless chat across language barriers.
- High-level summary of chosen technologies (React, Firebase, WebSockets, Microsoft Translator).

## 2. System Design & Architecture
- Overall architecture (frontend, backend, database, translation API).

- Guest users (ephemeral sessions).

- WebSocket setup and message flow.
- Typing indicators and presence tracking.
- Latency performance (<200 ms).

### 3.3 Language Translation
- Integration with Microsoft Translator API.
- Handling of original + translated text.

### 3.4 Data Retention & Privacy
- Firestore persistence for registered users.
- Automatic purge for guest sessions.

### 3.5 User Interface & Experience
- Responsive design (desktop/mobile).
- Profile management (avatar, display name, language).
- UI polish (auto-scroll, indicators, cohesive theme).
